# Гайд по ручному управлению контентом

Всё содержимое приложения (статьи, текст и настройки) компилируется из JSON-файлов в папке ресурсов. Это позволяет вам править контент напрямую без переписывания Kotlin-кода. Главный файл: `app/src/main/assets/data/articles.json`.

## 1. Как искать и переводить статьи вручную

1. Откройте в редакторе (Cursor/VSCode) файл `app/src/main/assets/data/articles.json`.
2. Используйте поиск (`Cmd + F`), чтобы найти статью по английскому названию (например, *"Survival Psychology"*).
3. Замените английский текст внутри поля `"content": "..."` на ваш перевод. Следите за тем, чтобы не сломать JSON-разметку (не удалите случайно запятые или кавычки).
4. Для применения изменений в приложении вам нужно каждый раз повышать версию базы данных: откройте файл `app/src/main/java/com/survivalwiki/data/AppDatabase.kt` и увеличьте значение `version = ...` на единицу (например, с `15` на `16`), затем сделайте билд.

## 2. Как и где заменять изображения и вставлять новые

1. **Расположение:** Все картинки лежат в папке `app/src/main/assets/images`.
2. **Добавление новой картинки:** Просто скопируйте любой `.jpg` или `.png` файл в эту папку.
3. **Вставка в текст:** Чтобы картинка появилась в статье, добавьте в поле `"content"` (в `articles.json`) специальный тег на отдельной строке:
  `<image:название_вашей_картинки.jpg:Ваша подпись под фото>`
   Например: `<image:fig03-01.png:Разведение огня трением>`

## 3. Как добавлять новые статьи вручную

1. Откройте `articles.json`.
2. Скопируйте любой существующий объект статьи (от `{` до `}`) и вставьте его в конец списка (или в нужную часть массива).
3. **Обязательно измените:**
  - `"id"`: задайте уникальный номер (например, 33, если было 32 статьи).
  - `"categoryId"`: укажите ID категории, к которой она относится (категории можно посмотреть в соседнем файле `categories.json`).
  - `"title"`, `"content"`, `"tags"`: заполните по смыслу.
  - `"iconResName"`: укажите иконку (например, `ic_build`, `ic_house`, `ic_medical_services`).
4. Как и при переводе, не забудьте после этого увеличить версию базы данных (`version = ...`) в `AppDatabase.kt` и пересобрать приложение!

## 4. Как назначать уровень сложности вручную

1. В файле `articles.json` у каждой статьи есть поле `"level"`.
2. Для изменения сложности просто отредактируйте это поле у конкретной статьи. Используйте одну из трех фраз:
  - `"level": "Уровень: Базовый"` (Отобразит 1 зелёный кружок в каталоге)
  - `"level": "Уровень: Средний"` (Отобразит 2 желтых кружка)
  - `"level": "Уровень: Продвинутый"` (Отобразит 3 красных кружка)
3. В самой статье эта фраза будет отображаться текстом "Уровень: Базовый", а на карточках в Каталоге и на Главной экране — в виде цветных кружков. Если вписать туда любой другой текст, кружки на карточках отображаться не будут, но текст в статье останется.
